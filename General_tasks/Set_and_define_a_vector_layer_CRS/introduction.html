As you can see in the map canvas, the Wake_BlockGroup_2010 layer seems shifted relatively to the rest of the project. This kind of issues is usually a sign that the layer's Coordinate Reference System (CRS) may be wrong or incomplete (that is, lacking the transformation parameters). We need to fix it before we can use the layer correctly.

There are two ways of fixing the problem:

  • Setting the correct CRS within the project.
  • Defining the layer's CRS in the source file.

We will start by setting the correct CRS within the project.

When ready, click Next step.
